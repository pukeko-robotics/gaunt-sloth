# v2.0.0 Lean Agent by Default

## New Features
- **Lean agent is now the default backend.** The plain LangChain agent (`GthLangChainAgent`) is the
  default for the CLI (`code`/`chat`), single-shot (`ask`/`exec`), and the AG-UI/`api` server. It runs
  with Gaunt Sloth's own toolset — filesystem, hardened dev/shell, and the new checklist tool. Set
  `agent.backend` to override per config.
- **`gth_checklist` planning tool.** The lean agent gains a first-party checklist/todo tool (the
  `write_todos` equivalent) for planning multi-step work and showing progress. Send the whole list
  each call; it replaces the previous one, keeps at most one item `in_progress`, and renders as a live
  `📋 Checklist` checkbox panel in the TUI. Enabled by default via `builtInTools: ["gth_checklist"]`.

## Potentially Breaking Changes
- **Default agent backend changed from `deep` to `lean`.** Sessions that relied on the implicit
  deepagents runtime (subagents, `write_todos`, summarization, `/large_tool_results` offload) must now
  opt in explicitly with `"agent": { "backend": "deep" }`. The deep backend is **experimental** and
  prints a warning when selected. The ACP server is unchanged (structurally deepagents-based).
- **`builtInTools` now defaults to `["gth_checklist"]`.** Previously no built-in tools were enabled by
  default. Setting your own `builtInTools` replaces this set entirely — include `gth_checklist` to keep
  it.

## Improvements
- **Quieter TUI.** `INFO`/`DEBUG` status lines (`Requested tools`, `Loaded tools`, `Workdir`, `Model`,
  `Thinking…`) are suppressed in the TUI, where they duplicated the live tool-call cards and status
  bar. `WARNING`/`ERROR` still surface; plain (non-TUI) CLI output is unchanged.
- **Documentation:** `docs/CONFIGURATION.md` documents `agent.backend` and `builtInTools`;
  `docs/ux-guidelines.md` documents the checklist panel and the TUI status-line suppression.
